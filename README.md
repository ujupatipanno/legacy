# 레거시 파일 플러그인

Obsidian에서 문서의 버전 이력을 관리하는 플러그인입니다. 현재 문서의 스냅샷을 레거시 파일로 저장하고, 여러 레거시 파일을 하나의 문서로 보존할 수 있습니다.

## 주요 기능

### 1. 레거시 파일 만들기
현재 작업 중인 문서의 사본을 타임스탬프와 함께 저장합니다.

- **파일명 형식**: `{문서명}_legacy_YYYYMMDDHHmmss.md`
- **저장 위치**: 설정에서 지정한 폴더 (기본값: `legacy`)
- **아이콘**: 💾 (lucide-save)

**사용 방법**:
- 명령어 팔레트에서 "현재 문서의 레거시 파일 만들기" 실행
- 또는 리본 메뉴의 💾 아이콘 클릭

### 2. 레거시 파일 보존하기
현재 문서와 관련된 모든 레거시 파일을 하나의 문서로 통합합니다.

- **통합 규칙**:
  - 각 레거시 파일 앞에 `#{문서명}_legacy_YYYYMMDDHHmmss` 태그 추가
  - 파일 사이에 `___` 구분선 삽입
  - 설정에 따라 오름차순 또는 내림차순 정렬
- **파일명 형식**: `{문서명}_legacy_{시작일시}-{종료일시}.md`
- **자동 정리**: 통합 완료 후 개별 레거시 파일 자동 삭제
- **아이콘**: 💾 (lucide-save-all)

**사용 방법**:
- 명령어 팔레트에서 "현재 문서의 레거시 파일 보존하기" 실행
- 또는 리본 메뉴의 💾 아이콘 클릭

## 설정

플러그인 설정 탭에서 다음 항목을 구성할 수 있습니다:

### 레거시 폴더 위치
레거시 파일이 저장될 폴더 경로를 지정합니다.
- **기본값**: `legacy`
- **예시**: `archive/legacy`, `backup` 등

### 정렬 순서
레거시 파일을 보존할 때 정렬 순서를 선택합니다.
- **오름차순**: 오래된 것부터 최신 순서로 정렬
- **내림차순**: 최신 것부터 오래된 순서로 정렬

## 사용 예시

### 시나리오: 문서 작업 이력 관리

1. **작업 중 스냅샷 저장**
   ```
   문서: "프로젝트 계획서.md"
   레거시 파일 만들기 실행
   → legacy/프로젝트 계획서_legacy_20251215103045.md 생성
   ```

2. **여러 번 저장**
   ```
   legacy/프로젝트 계획서_legacy_20251215103045.md
   legacy/프로젝트 계획서_legacy_20251215143022.md
   legacy/프로젝트 계획서_legacy_20251215173512.md
   ```

3. **레거시 파일 통합**
   ```
   레거시 파일 보존하기 실행
   → legacy/프로젝트 계획서_legacy_20251215103045-20251215173512.md 생성
   → 개별 레거시 파일 3개 자동 삭제
   ```

4. **통합 파일 구조**
   ```markdown
   #프로젝트 계획서_legacy_20251215103045
   
   [첫 번째 버전 내용]
   
   ___
   
   #프로젝트 계획서_legacy_20251215143022
   
   [두 번째 버전 내용]
   
   ___
   
   #프로젝트 계획서_legacy_20251215173512
   
   [세 번째 버전 내용]
   ```
